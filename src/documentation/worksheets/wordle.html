Preliminaries

  • If you are not familiar with Wordle puzzles, then before proceding, visit the official online Wordle site, read the instructions and try to solve today's puzzle.
  • Set the browser to Full Screen view, to give this worksheet, and your code, as much space as possible.
  • Remember that if the code editor has a tinted background it does not have focus and will not respond to edits. To give it focus click within the editor. (If a program is running, you will need to stop it first.)
  • Use the auto-completion options as much as possible, because this reduces errors.
  • Use the Hints only when you need them - as all use of hints is recorded. Use of Help is encouraged and not recorded.

Step 1

  • Select File > Load and locate the file wordle-start.elan. Select File > Auto Save and save the file to your personal file system. This will ensure that the changes you will be making to your version of the code are always saved.
  • Run the game and the Display will show a re-creation of the Wordle screen. Try entering your first and second guess words. You will notice that any word you enter receives the identical response - and as you enter more words those identical responses will sooner or later become inconsistent with each other.
  • Stop the program and explore the code. Look first at frame number XX - the function named markAttempt. A comment in the code explains that this is a 'stub' - a minimal implementation that lets the program compile and run, but which is known not to be correct. Your first task is to make the Wordle game work by writing a proper implementation of this function.
  • Immediately below this function you will find a test, which is 'ghosted' (indicated by the # [ghosted] 'compiler directive, and by the pale tint') so that it is currently ignored by the compiler). Select the test, then from the 'context menu' (brought up on a selected instruction by right-mouse-click, or by pressing Ctrl-m) and selecting unghost.
  • The Test status will now indicate that all of the assert instructions are failing.
  • Examine these individual assert statements noting each time markAttempt is called, the first argument is the attempt (guess) and the second is the target (the word that has been set as the answer. The function should return a string of five characters, each of which must be 0to represent 'no match' (grey), 1, to represent an 'out of place' match (yellow), or 2 to represent an 'in-place' match (green). These three numeric digits are translated into the colours to be rendered on the Display by the style sheet, which may be found in frame number xx.
